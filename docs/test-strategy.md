# API Test Strategy

## Project
Conduit API Testing

## Objective
The purpose of this project was to validate selected REST API endpoints of the Conduit application using Postman.

## Scope
The following areas were covered:
- user registration
- user login
- current user retrieval
- article creation
- article update
- article deletion

## Test Types
The following test types were used:
- positive testing
- negative testing
- response validation
- status code validation
- authentication testing

## Validation Areas
The following elements were checked:
- correct status codes
- response body structure
- required fields
- authentication behavior
- error handling for invalid data

## Tools
- Postman
- JSON
- GitHub

## Result
The collection was created to verify the correctness of the most important API flows in the Conduit application.
